Xcode注释标识符

［Xcode］

前几天总结了一下Xcode注释的问题，因为大家给自己代码加注释的时候，

无非单行注释// ，多行注释用/**/,或者使用插件例如VVDocument等

等。实际上，给代码加注释的时候,我们可以给注释加上一些标识符，这些标识

符能够被Xcode识别，起到类似于＃pragma mark的作用。

// TODO: 0000 (我们将要做的事情)

// MARK: 1111 (我们标记了某行代码)

// FIXME: 2222 (我们要修改的代码) 

// ???: 3333 (可能有疑问的代码)

// !!!: 4444 (可能要注意的代码)

大家使用的时候要注意一下，必须严格按照上面的格式,冒号后面一定要有内容,
