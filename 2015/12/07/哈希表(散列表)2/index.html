一 哈希函数的构造方法

哈希哈数主要有这么几种构造方法,直接定址法、数字分析法、平方取中法、除留取余法、 折叠法等……(这里我们先主要说一下直接定址法和除留取余法).

哈希函数的构造原则:1 计算简单,这样不会降低查找的效率. 2 哈希地址均匀分布,保证存储空间的充分利用.

直接定址法:

如果我们要对0-100岁的人口数字进行统计,那么我们年龄这个关键字就可以用年龄的数字作为哈希地址,即 f(key) = key;

这个时候, 我们可以得出这么个哈希函数:f(0) = 0, f(1) = 1, …… ,f(20) = 20,这就是直接定址法.

如果我们现在要统计的是80后出生年份的人口数, 那么我们对出生年份这个关键字可以用年份减去1980来作为地址. 此时f (key) = key-1980.
假如今年是2000年，那么1980年出生的人就是20岁了, 此时 f(2000) = 2000 - 1980, 可以找得到地址20, 地址20里保存了数据”人数1500万”.

也就是说, 我们可以取关键字的某个线性函数值为哈希地址, 即: f(key) = a * key + b, 这样的哈希函数优点就是简单, 均匀, 也不会产生冲突, 但问题是这需要事先知道关键字的分布情况, 适合査找表较小且连续的情况.

除留取余法:

除留取余法是最常用的哈希函数构造方法. 对于哈希表长为m的哈希函数公式为:f(key) = key mod p (p ≤ m). 事实上, 这个方法不仅可以对关键字直接取

模, 也可在通过折叠法, 平方取中法之后再取模(mod).

除留取余法关键就在于选择合适的p, p如果选得不好，就可能会容易产生哈希碰撞(同义词)。

假设有一张表，它有9条记录，现在我们要针对它设计一个哈希表。如果采用除留余数法，那么可以先尝试将哈希函数设计为f(key) = key mod 9的方法。比如

29 mod 9 = 2，所以它存储在下标为2的位置。

f( key ) = key mod p ( p ≤ m ), p取值越接近m越好, 且最好是一个质数.

二: 解决哈希碰撞问题

从前面设计哈希函数的方法可以看出, 难免会出现哈希碰撞的问题, 我们主要有以下几种方法去处理: 开放地址法、再哈希法、链地址法、建立公共溢出区法等, 在这里我们主要研究开放地址法和链地址法.

开放地址法:

开放定址法就是一旦发生了碰撞, 就去寻找下一个空的哈希地址, 只要哈希表足够大, 空的哈希地址总能找到, 并将记录存入.
公式为:f(key) = (f(key)+d) MOD m (d=1,2,3,……,m-1)
比如说, 我们的关键字集合为{12,67,56,16,25,37,22,29,15,47,48,34},表长为12. 我们用哈希函数f(key) = key mod l2.
当计算前S个数{12,67,56,16,25}时, 都是没有冲突的哈希地址, 直接存入：

当key ＝ 37 时, f(37) = 1, 就与25所在位置产生碰撞, 于是我们带入公式: f(key) = (f(key)+d) MOD m (d=1,2,3,……,m-1), f(37) = (f(37)+1) mod 12 = 2, 这时候, 我们就把37存在2的位置.

接下来22,29,15,47都没有冲突, 正常的存入, 到了 key=48, f(48) = 0, 与12所在的0位置碰撞, 带入公式f(48) = (f(48)+1) mod 12 = 1, 此时又与25所在的位置碰撞. 一直到 f(48) = (f(48)+6) mod 12 = 6时, 才能存入哈希表.

链地址法:

将所有关键字为同义词的记录存储在一个单链表中，我们称这种表为同义词子表，在哈希表中只存储所有同义词子表的头指针。

链地址法(拉链法)解决碰撞的做法是: 将所有关键字为同义词的结点链接在同一个单链表中. 若选定的散列表长度为m, 则可将散列表定义为一个由m个头指针组成的指针数组T[0..m-1].

凡是散列地址为i的结点, 均插入到以T[i]为头指针的单链表中. T中各分量的初值均应为空指针. 在拉链法中, 装填因子α可以大于 1, 但一般均取α≤1.
