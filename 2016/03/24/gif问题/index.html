一个关于iPhone本地相册保存的gif,在自己app种读取gif的问题

［iOS］

前段时间有个朋友问我app想实现这样一个功能,读取iPhone本地相册中,从网络上下载的gif图片,或者从别的

app中保存下来的gif图片.

大家都知道iPhone本地相册不支持播放gif的,它只显示第一帧的静态图片,但是从网络上保存下载的图片的

data,依然也被保存到相册中了, 所以我们需要把data读取并转化成image即可.

iOS8之后,photokit框架替代了之前的AlAssetLibrary库,供开发者使用.

假设我最新保存了一张gif到本地相册,如何让我的app中的imageView显示gif呢?

// 选中照片的代理方法 
-(void)imagePickerController:(UIImagePickerController *)picker didFinishPickingMediaWithInfo:(NSDictionary<NSString *,id> *)info{
PHAsset *asset = nil; // 照片信息
PHFetchOptions *fetchOptions = [PHFetchOptions new]; // 获取信息的相关选择
fetchOptions.sortDescriptors = @[[[NSSortDescriptor alloc] initWithKey:@"creationDate" ascending:YES]]; // 使用照片的创建时间,升序,作为排序依据.


PHFetchResult *result = [PHAsset fetchAssetsWithMediaType:PHAssetMediaTypeImage options:fetchOptions]; // 获取结果类型为image类型

if (result != nil && result.count > 0) {
    asset = [result lastObject]; // 获取最新一张照片
}

if (asset) {
    PHImageRequestOptions *op = [PHImageRequestOptions new];
 //   op.synchronous = YES;

    [[PHImageManager defaultManager] requestImageDataForAsset:asset options:op resultHandler:^(NSData * _Nullable imageData, NSString * _Nullable dataUTI, UIImageOrientation orientation, NSDictionary * _Nullable info) {

        dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
            NSLog(@"%@",info);
            NSURL *url = info[@"PHImageFileURLKey"];

            NSURLSessionDataTask *dt = [[NSURLSession sharedSession] dataTaskWithURL: url   completionHandler:^(NSData * _Nullable data, NSURLResponse * _Nullable response, NSError * _Nullable error) {

                NSLog(@"%@",error);
 dispatch_async(dispatch_get_main_queue(), ^{

                self.myImage.image = [UIImage imageWithData:data];

                });

            }];

            [dt resume];

        });


    }];

}

[self dismissViewControllerAnimated:YES completion:nil];

}

就这样,我们就把最新保存下来的gif给获取到了.
