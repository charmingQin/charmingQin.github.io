一 哈希函数的构造方法

哈希哈数主要有这么几种构造方法,直接定址法、数字分析法、平方取中法、除留取余法、 折叠法等……(这里我们先主要说一下直接定址法和除留取余法).

哈希函数的构造原则:1 计算简单,这样不会降低查找的效率. 2 哈希地址均匀分布,保证存储空间的充分利用.

直接定址法:

如果我们要对0-100岁的人口数字进行统计,那么我们年龄这个关键字就可以用年龄的数字作为哈希地址,即 f(key) = key;

/Users/qinxiaoqiang/Desktop/屏幕快照 2015-12-05 上午10.06.09.png

这个时候, 我们可以得出这么个哈希函数:f(0) = 0, f(1) = 1, …… ,f(20) = 20,这就是直接定址法.

如果我们现在要统计的是80后出生年份的人口数, 那么我们对出生年份这个关键字可以用年份减去1980来作为地址. 此时f (key) = key-1980.
假如今年是2000年，那么1980年出生的人就是20岁了, 此时 f(2000) = 2000 - 1980, 可以找得到地址20, 地址20里保存了数据”人数1500万”. 

/Users/qinxiaoqiang/Desktop/屏幕快照 2015-12-05 上午10.16.43.png

也就是说, 我们可以取关键字的某个线性函数值为哈希地址, 即: f(key) = a * key + b, 这样的哈希函数优点就是简单, 均匀, 也不会产生冲突, 但问题是这需要事先知道关键字的分布情况, 适合査找表较小且连续的情况.

除留取余法:

除留取余法是最常用的哈希函数构造方法. 对于哈希表长为m的哈希函数公式为:f(key) = key mod p (p ≤ m). 事实上, 这个方法不仅可以对关键字直接取

模, 也可在通过折叠法, 平方取中法之后再取模(mod).

除留取余法关键就在于选择合适的p, p如果选得不好，就可能会容易产生哈希碰撞(同义词)。

假设有一张表，它有9条记录，现在我们要针对它设计一个哈希表。如果采用除留余数法，那么可以先尝试将哈希函数设计为f(key) = key mod 9的方法。比如

29 mod 9 = 2，所以它存储在下标为2的位置。

如果还是有9条记录，关键字如下: 

/Users/qinxiaoqiang/Desktop/屏幕快照 2015-12-05 上午11.05.15.png

那这样 18、27、36、63等mod 9 都等于0, 这样取余数并不好, 碰撞比较多, 我们可以选择mod 7 来进行除留取余法. 所以对于除留取余法这个哈希函数,

f( key ) = key mod p ( p ≤ m ), p取值越接近m越好, 且最好是一个质数.
