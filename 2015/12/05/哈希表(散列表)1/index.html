一:什么是哈希表(HashTable)

哈希表,也叫散列表,是一种以键值对(key-value)存储数据的数据结构.

二:哈希表的构建思路

有过语言基础的读者都知道数组这种数据类型,比如我用C语言定义一个整型数组:

int arr[3] = {0,1,2};

数组每个元素的下标是0,1,2(也就是key值);对应的值是0,1,2(也就是value值);这个时候我们可以认为数组

下标和数组元素的值,有对应关系,也就是说,数组就是一种特殊的,简单的哈希表.

但是此时,key值是整数,value是有序的,如果key值是字符串,value是无序的,那我们就要使用哈希表这种数据

结构了.

三:一道经典哈希表面试题

我们可以先通过一道面试题对哈希表有一个初步的认识.

如果从一个足够长的字符串中，找出第一个唯一的字符，应该如何查找？如输入abaccdeff,则输出b.(注：这是2006年google的一道笔试题.)

分析:由于字符串是字符(char)构成的，而char类型总共只有总共只有256种可能，我们可以创建一个长度为

256的数组，每个字符根据对应的ASCII码值作为数组的下标，数组里面存着字符出现的次数，这样我们就

创建了一个大小为256，以ASCII码为值的哈希表。

// 创建一个参数为输入的字符串，返回值为第一个唯一字符的方法。
 char FindTheUniqueChar(char* String)
{
// 如果输入错误，直接返回
    if(!String){
    printf("您的输入有误，请重新输入！");
    return 0;
    }

// 初始化一个哈希表，大小为256
   const int tableSize = 256;
   unsigned int hashTable[tableSize];
   for(unsigned int i = 0; i < tableSize; i++){

// 获得字符串中每一个字符出现的次数
     char* hashTableKey = String;
     while(*(hashTableKey) != '\0')
     hashTable[*(hashTableKey++)] ++; //以字符的数值大小为哈希数组的下标

// 找到字符串中只出现一次的字符
     hashTableKey = String;
     while(*hashTableKey != '\0')
     {
        if(hashTable[*hashTableKey] == 1)
          return *hashTableKey;

        hashTableKey++;
     }
   }

return 0;

}

由此可以看出,如果没有内存限制，我们将key作为数组的索引，那么所有的查找时间复杂度为O(1)，

即一次读取就能找出唯一的字符，所以哈希表的查找效率确实可见一斑。

四:通过上面的面试题,我们分析一下哈希表的实现步骤.

既然哈希表是通过key-value形式存储的数据,我们就可以通过key,获取到对应的value,我们刚才就是将字符的

ASCII码值作为key,value里面是放的是字符出现的次数,也就是说先确定出容器的索引,然后对应的存储

需要的value.所以哈希表的实现步骤大致总结为以下两步:

1 分析我们要得到的结果,使用一个映射方法,获取到我们要使用的key,这个函数叫做哈希函数.前人(应该是一帮数学家

或者密码学家)帮我们总结了一些哈希函数的构造方法.

2 实际的数据模型可能不会key-value一一对应,有可能一个key对应多个value,这种情况叫哈希碰撞(冲突),我们要尽量

避免冲突.
